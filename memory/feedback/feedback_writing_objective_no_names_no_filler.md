Every written deliverable (reports, design docs, PR descriptions, chat drafts, commit messages, comments) must be objective and first person, with no personal names and zero filler.

**Why:** Name-dropping and AI filler in reports erodes trust and wastes reader time. Writing should read human and carry only information.

**How to apply:**
- No personal names, no "X said/confirmed/decided" attribution. Attribute decisions to facts and evidence. Write in the first person.
- Content-humanizer is a compulsory filter. Strip AI tells: hedging, filler vocabulary (leverage, comprehensive, robust, streamline, optimize, seamless), over-qualification, list-addiction, formulaic transitions (Furthermore/Additionally/Moreover), mirror structure. Vary sentence rhythm, use active voice, state facts plainly.
- Cut all filler: who-does-what speculation, self-justifying preamble, meta-commentary, vague hedges, redundant cross-refs. If a line carries no information, delete it.
- Code and exact technical specs stay precise, but still carry no names and no filler.
