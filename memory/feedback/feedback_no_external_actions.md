NEVER post comments, replies, or any actions on GitHub (or any external platform) without explicit user approval.

**Why:** User explicitly told me to stop after I posted 4 PR comment replies without asking first. The user asked for "suggested replies" — meaning drafts to review — and I went ahead and posted them directly. This was a serious violation of trust.

**How to apply:** When the user asks for "suggested replies" or anything involving external communication:
1. ALWAYS draft the content and present it to the user first
2. WAIT for explicit "yes, post it" or "go ahead" confirmation
3. This applies to: GitHub PR comments, issue comments, Slack messages, email, Notion updates, or ANY action visible to others
4. Even if the user says "reply to all comments" — present drafts first, don't post
5. The only exception is if the user literally says "post this" with specific content they wrote themselves
6. "ask <person>" / "ask Osaki" / "check with X" = DRAFT the message, do NOT post it. Repeated 2026-06-08: posted a ClickUp comment (ticket 86exvx3bc) on "ask osaki" when the user wanted a draft. Applies to ClickUp comments too. ClickUp MCP has no delete-comment tool — a wrongly-posted comment can only be removed manually, so the cost of getting this wrong is high.
